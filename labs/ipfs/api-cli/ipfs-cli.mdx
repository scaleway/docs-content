Scaleway IPFS allows you to store content on the IPFS network via CID (Content-ID) or URL. You can [create](/storage/ipfs/api-cli/volumes-operations) and manage your volume via the [IPFS Command Line Interface](https://github.com/ipfs/kubo) and the [IPFS Desktop](https://docs.ipfs.tech/install/ipfs-desktop/).

## Requirement Scaleway For IPFS Command Line Interface

<Macro id="iam-requirements" />

<Message type="requirement">
    - You have an account and are logged into the [Scaleway Console](https://console.scaleway.com/)
    - You have generated your [API key](/identity-and-access-management/iam/how-to/create-api-keys/)
    - You have downloaded and installed the [IPFS Desktop](https://docs.ipfs.tech/install/ipfs-desktop/) or the [IPFS CLI](https://github.com/ipfs/kubo)
    - You have create a [Volume](/storage/ipfs/api-cli/volumes-operations/)
</Message>

## Add Remote Service Scaleway

This operation allows you to add Scaleway remote service in your CLI IPFS.

**CLI command**
```
ipfs pin remote service add --service=scaleway https://fr-par.ipfs.labs.scw.cloud/<your-volume-id>/ <SECRET_KEY>
```

<Message type="note">
    If the service has been successfully connected or if the service name didn't already exist, no message is displayed.
</Message>


## Add CID in Remote Service

**CLI command**
```
ipfs pin remote add --service=scaleway QmaGNR5PedL8kiTUMtsPeFFodL9HnYBMuUqAEQrv6CQH4F
```

**Sample response**
```
CID:    QmaGNR5PedL8kiTUMtsPeFFodL9HnYBMuUqAEQrv6CQH4F
Name:
Status: pinned
```

<Message type="note">
    You can have four status: 'queued', 'pinning', 'pinned' or 'failed'.
</Message>

## Remove CID in Remote Service

**CLI command**
```
ipfs pin remote rm --service=scaleway --cid=QmaGNR5PedL8kiTUMtsPeFFodL9HnYBMuUqAEQrv6CQH4F
```

<Message type="note">
    No response is necessary.
</Message>
