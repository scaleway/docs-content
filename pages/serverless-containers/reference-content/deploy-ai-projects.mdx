## Why Serverless Containers for AI projects

AI applications, often have unique traffic patterns and resource requirements. Serverless Containers addresses these needs:

- **Scale to zero**: Many AI projects like proof of concepts and more often have "bursty" traffic. With Serverless Containers only pay when your container running, when idle it scales to zero.
- **High-resource capabilities**: AI apps often need more than a standard function's memory. Scaleway supports up to 12 GB of RAM and 6 vCPUs per instance, allowing you to run local embeddings or heavy Python libraries.
- **Standardization**: Unlike restricted runtimes, containers allow you to use any language or version and install any OS-level dependency (like ffmpeg for audio processing).

## From code to container

The bridge between your AI editor and the cloud is the container. A container packages your code, its environment, and its dependencies into a single image.

For most AI-assisted developers, the workflow looks like this:

**The prompt**: You ask your AI agent to "Create an API backend for an image generator."
**The Dockerfile**: You ask the agent to "Generate a Dockerfile for this app." The AI handles the complexity of environment setup.
**The deployment**: You push that image to the Scaleway Container Registry and deploy it.

## How Serverless Containers works

The platform is designed to abstract away the infrastructure so you can focus on your application logic.

- **Namespaces**: These act as project folders. You can group related services (e.g., a frontend, a backend, and a database-sync worker) and share environment variables across them.
- **Containers**: This is the running instance of your code. Each container gets a unique, auto-generated URL secured.
- **Autoscaling**: Scaleway automatically creates new instances of your container as traffic increases and removes them when it drops.

## Optimizing for AI and vibe coding

To get the most out of your AI apps on Scaleway, follow these best practices:

### Managing secrets (API keys)

AI apps rely heavily on external providers.

- **Never hardcode keys**: Use Secret Environment Variables in the Scaleway console.
- Secrets are encrypted and injected into your container at runtime, keeping your project secure.

Refer to [How to secure a container](/serverless-containers/how-to/secure-a-container/) for detailed instructions on managing environment variables.

### Handling cold starts

When a container scales to zero, the first request after a break takes a few seconds to "warm up."

<Message type="tip">
Use lightweight base images (e.g., `python:3.11-slim`) to speed up startup.
</Message>

<Message type="tip">
If your app requires instant response times (e.g., a public-facing chat UI), set `min_scale=1`.
</Message>

### Monitoring with Cockpit

Use [Scaleway Cockpit](/cockpit/) to monitor your AI app's health. You can track memory usage (crucial for LLM logic) and view logs in real-time to debug issues that didn't appear during local development.

## Quickstart: Deploy your AI project

If you have a Python app ready, your Dockerfile should look something like this:

```dockerfile
FROM python:3.11-slim

# Set environment variables
ENV PYTHONUNBUFFERED=1
WORKDIR /app

# Install dependencies
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

# Copy application code
COPY . .

# Start the application
# Serverless Containers injects a PORT variable; ensure your app listens on it
CMD ["sh", "-c", "uvicorn main:app --host 0.0.0.0 --port ${PORT:-8080}"]
```

1. **Push your image**: Use `docker push` to send your image to your Scaleway Container Registry namespace.
2. **Deploy**: In the Scaleway Console, select your image and click **Deploy**.
3. **Connect**: Use the provided endpoint to start interacting with your AI application.

Refer to [How to deploy a container](/serverless-containers/how-to/deploy-container/) for detailed deployment instructions.
