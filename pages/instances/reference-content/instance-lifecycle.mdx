The lifecycle of Scaleway Instances consists of two distinct layers: the **operational lifecycle** (the state transitions of an individual resource) and the **product-level lifecycle** (the availability and retirement of specific hardware generations).

### 1. Operational lifecycle of an individual Instance

This covers the journey of a single Instance from creation to deletion. These states are manageable via the Scaleway Console, API, CLI, or Terraform.

#### Key states / Power modes
| State | Description |
| --- | --- |
| **Running (Started)** | The Instance is fully active, consuming compute resources. This is the default after creation or power-on. Full billing applies (compute + storage + IPs). |
| **Standby** | The Instance is halted (powered down) but remains allocated to your account. Data stays on local storage for fast restart. Quick to resume, but **billed exactly like a running Instance**. Use this for short-term pauses. |
| **Powered Off (Stopped)** | All local-volume data is automatically archived to a volume store; the physical hypervisor node is released back into Scaleway’s pool. Compute billing stops (you still pay for any Block Storage volumes and Flexible IPs). Restarting restores the data automatically. This is the recommended state for long-term cost savings. |
| **Error / Locked / Protected** | Temporary or administrative states (e.g., during resize, rescue mode, or when you enable protection to prevent accidental stop/delete). |
| **Terminated (Deleted)** | The Instance and its local volumes are permanently removed. Flexible IPs attached to it are preserved unless you release them manually. |


### 2. Product-Level Lifecycle of Instances (Hardware Generations)

Scaleway regularly refreshes hardware to ensure performance and sustainability. When a specific Instance type (e.g., an older generation) reaches the end of its cycle, it follows these stages:

#### Key states
| State | Description |
| --- | --- |
| **General Availability (GA)** | Fully launched and supported. Covered by the standard SLA. |
| **End of Deployment** | No new hardware is added to the pool. The offer still receives features, patches, and bugfixes. |
| **End of New Features** | No new features are developed. The offer only receives critical security patches and bugfixes. |
| **End of Growth** | The Instance type can no longer be ordered by **new** customers. Existing customers can still create new instances. |
| **End of Sales** | The Instance type can no longer be ordered by **any** customer. Existing instances continue to run normally. |
| **End of Life (EOL)** | The service is retired. Any remaining Instances are forcibly shut down. |

<Message type="tip">
  Scaleway will **inform you via email and console notifications** once an Instance generation enters its **End of Sales** period, providing you with ample time to plan your migration to a newer generation before the End of Life date.
</Message>
