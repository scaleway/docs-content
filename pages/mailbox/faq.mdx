## Overview

### What is Mailbox?

Mailbox is a professional email service allowing you to create and manage email accounts using your own domain.
It supports standard email protocols such as IMAP and SMTP, allowing access from most email clients (webmail, desktop or mobile applications).

Mailbox also supports calendar and contact synchronization using [CalDAV](/mailbox/concepts/#calendar-caldav) and [CardDAV](/mailbox/concepts/#contacts-carddav).

### How much storage is included per Mailbox email account?

Each Mailbox email account includes **20 GB of storage** shared across:
  - emails
  - attachments
  - mailbox folders

### What happens if my mailbox reaches its storage limit?

If a mailbox reaches its 20 GB storage limit, it cannot receive new emails until space is freed.
To restore normal operation, you can delete emails and attachments, or archive messages locally.

### Can I use Mailbox to send transactional emails?

No. Mailbox is designed for human communication, such as professional or personal email exchanges.
If you need to send automated emails from an application (account confirmation, password reset, notifications, OTP, etc.), we recommend using **[Transactional Email (TEM)](/transactional-email/)**.

### Can I use Mailbox to send newsletters or marketing emails?

No. Mailbox is not intended for bulk or marketing emails such as newsletters or promotional campaigns.

For these use cases, we recommend using dedicated email marketing platforms.

### How many mailboxes can I create?

During the Beta phase, you can create **up to 20 mailboxes per project**.
If you need to create more mailboxes, please contact the [Scaleway Support team](https://console.scaleway.com/support/tickets/create).

<Message type="note">
  These limits may evolve when the product reaches General Availability (GA).
</Message>

### Can I use my own domain?

Yes. Mailbox allows you to use your own domain name.

You need to configure the required DNS records (such as MX records) so that emails can be routed to your mailboxes.

### Do I need to configure DNS records manually?

It depends on where your domain is managed.
- If your domain is managed by Scaleway, an Autoconfiguration feature sets the required DNS records for you.
- If your domain is managed outside Scaleway, you need to configure DNS records manually.

<Message type="note">
  The Autoconfiguration feature will be available before General Availability.
</Message>

### Which protocols are supported?

Mailbox supports standard email protocols:
- **IMAP** : receive emails
- **SMTP** : send emails
- **CalDAV** : synchronize calendars
- **CardDAV** : synchronize contacts

This allows Mailbox to work with most email clients.

### Can I access my mailbox from any email client?

Yes. Mailbox can be used with most email clients supporting IMAP and SMTP, including:
- Apple Mail
- Microsoft Outlook
- Thunderbird
- Other mobile email client applications

### Can I access my mailbox through webmail?

Yes. Mailbox includes a webmail interface allowing you to access emails, calendars, and contacts directly from a web browser.

### Can I migrate emails from another provider?

Mailbox migration tools are not yet available in the Beta phase.
At the moment, mailboxes must be created manually and used as new email accounts.

Migration tools will import emails, calendars, and contacts from other providers (such as OVH, Gandi, or other IMAP-compatible services).

<Message type="note">
  Mailbox migration tools are planned for after the General Availability (GA) of the product.
  In the meantime, if you need to migrate existing data, you may use third-party migration tools compatible with IMAP, CalDAV, and CardDAV.
</Message>

### What are the current Beta phase limitations?

In the Beta phase:
- mailbox storage is limited to 20 GB per mailbox.
- you can create up to 20 mailboxes per project.
- some features may still evolve before General Availability.

Feedback from Beta users helps improve the product before General Availability (GA).
