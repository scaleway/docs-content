The `scw container deploy` command is a Scaleway CLI command that automates the entire deployment pipeline for Serverless Containers in a single step. It handles namespace creation, image building (via Docker or Buildpacks), pushing to the registry, and deploying the container — automatically.

<Requirements />

- A Scaleway account logged into the [console](https://console.scaleway.com)
- [Owner](/iam/concepts/#owner) status or [IAM permissions](/iam/concepts/#permission) allowing you to perform actions in the intended Organization
- A valid [API key](/iam/how-to/create-api-keys/)
- [Installed and configured the Scaleway CLI](/scaleway-cli/quickstart/)
- **Docker** installed and running locally (required even for Buildpacks-based builds)
- An active Scaleway project with Container and Registry quotas

## Syntax

```bash
scw container deploy [arg=value ...]
```

## Arguments

| Argument           | Default                                      | Description                                                |
| ------------------ | -------------------------------------------- | ---------------------------------------------------------- |
| `name`             | Name of the `build-source` folder            | Application name                                           |
| `builder`          | `paketobuildpacks/builder-jammy-base:latest` | Builder image to use                                       |
| `run-image`        | `paketobuildpacks/run-jammy-base:latest`     | Run image to use                                           |
| `dockerfile`       | `Dockerfile`                                 | Path to the Dockerfile                                     |
| `force-builder`    | `false`                                      | Force Buildpacks builder even when a Dockerfile is present |
| `build-source`     | `.`                                          | Path to the build context                                  |
| `cache`            | `true`                                       | Enable build caching                                       |
| `build-args.{key}` | —                                            | Environment variables at build time                        |
| `port`             | `8080`                                       | Port to expose                                             |
| `namespace-id`     | —                                            | ID of the target Container namespace                       |
| `region`           | `fr-par`                                     | Target region (`fr-par`, `nl-ams`, `pl-waw`)               |

## Basic example

### Step 1: Prepare the application

Create a folder `my-app/` containing an `index.html` file and a `Dockerfile`:

```text
my-app/
├── index.html
└── Dockerfile
```

**index.html**:

```html
<!DOCTYPE html>
<html>
  <head>
    <title>My Application</title>
  </head>
  <body>
    <h1>Welcome to Scaleway!</h1>
  </body>
</html>
```

**Dockerfile**:

```dockerfile
FROM nginx:alpine
COPY index.html /usr/share/nginx/html/index.html
EXPOSE 80
```

### Step 2: Deploy

Run the following command in your terminal:

```bash
scw container deploy
```

### Expected output

The command performs the following operations automatically:

1. **Fetches or creates** the `my-app` namespace in the `fr-par` region
2. **Fetches or creates** the registry associated with the namespace
3. **Builds the image** via Docker (since a `Dockerfile` is present)
4. **Pushes the image** to the registry
5. **Creates or updates** the container with the new image
6. **Deploys the container** and waits until it is `ready`

```text
[1/6] Fetching namespace ...
[2/6] Fetching or creating registry ...
[3/6] Packing image ...
[4/6] Building image ...
[5/6] Pushing image ...
[6/6] Deploying container ...
  ✓ Container deployed successfully
  Name:       my-app
  ID:         xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx
  Region:     fr-par
  Image:      registry.fr-par.scw.my-app/xxxxx:latest
  Port:       80
  Status:     ready
```

## Redeploying

Running the same command again automatically updates the existing container.

Run the initial deployment command:

```bash
scw container deploy name=my-app build-source=./my-app port=80
```

After modifying the code, run the same command again to update the container:

```bash
scw container deploy name=my-app build-source=./my-app port=80
```

## Domain name

After deployment, the application is accessible at the following URL:

```text
https://<random-prefix>-<app-name>.functions.fnc.<region>.scw.cloud
```

Example:

```text
https://clitctnrdeploysnwevqmkl-cli-t-ctnr-deploy-s.functions.fnc.fr-par.scw.cloud
```

## Languages supported by Buildpacks

Buildpacks automatically detect the language and build the image **without requiring a Dockerfile**. The following languages are supported:

| Language    | Required files                                  |
| ----------- | ----------------------------------------------- |
| **Go**      | `go.mod` and `go.sum`                           |
| **Node.js** | `package.json` (optionally `package-lock.json`) |
| **Python**  | `requirements.txt` or `Pipfile`                 |
| **Java**    | `pom.xml` (Maven) or `build.gradle` (Gradle)    |
| **Rust**    | `Cargo.toml`                                    |
| **PHP**     | `composer.json`                                 |

## Internal workflow

The `scw container deploy` command follows this sequence:

1. **Fetch or create namespace**
2. **Fetch or create registry**
3. **Pack image** — Archives the build context
4. **Build image** — Docker build or Buildpacks (depending on Dockerfile presence)
5. **Push image** — To Scaleway Registry
6. **Create or update container** — With the new image
7. **Deploy container** — Deploy and wait for ready
