## Data Version Control (DVC) - Overview

Coucou alors https://dvc.org/ ça permet de gérer des uses case de data science

Dans l’ensemble git n’est pas adapté pour stocker plein de données très larges genre plusieurs gigas et c’est pour ça qu’on a recourt a l’utilisation d’un object storage

et typiquement dvc ne va pas deviner où aller avec l’object storage

Niveau post check si tu vérifies que tu as bien quelque chose dans ton object storage je pense

les étapes de la procédures:

If you installed DVC via pip and plan to use cloud services as remote storage, you might need to install these optional dependencies: [s3], [azure], [gdrive], [gs], [oss], [ssh]. Use [all] to include them all. For example:

`dvc get` allows to scrap any file or folder of a DVC or Git repo, turning them into data registry.

## Setting up DVC

<Macro id="iam-requirements" />
 
<Message type="requirement">
  - You have an account and are logged into the [Scaleway console](https://console.scaleway.com)
  - You have a running Object storage (link to doc)
  - You have a bucket (link to doc) private or public ?
  - You have dvc installed (link to dvc install doc)
  - You have [created an API key](https://www.scaleway.com/en/docs/identity-and-access-management/iam/how-to/create-api-keys/)
  - You have a repo (init empty repo or existing one)
  - You have done your [first request with scaleway API](https://www.scaleway.com/en/developers/api/#quickstart:-first-request)
  - You [authenticated](https://www.scaleway.com/en/developers/api/#how-to-authenticate) for the first time 
  - You have installed [AWS CLI](https://www.scaleway.com/en/docs/storage/object/api-cli/object-storage-aws-cli/)
</Message> 

1. `pip install dvc`
1. `pip install "dvc[s3]"`
1. `dvc init` in repo
2. `git add` and `git commit` dvc files
3. ```bash
   dvc get https://github.com/iterative/dataset-registry \
            get-started/data.xml -o data/data.xml
   ```
4. `dvc add data/data.xml` to stage files to store on remote
5. `git add data/data.xml.dvc data/.gitignore`
6. `git commit -m "Add raw data"`
7. `dvc remote add -d  myremote s3://mybucket/path`
8. `dvc remote modify myremote endpointurl https://my-bucket.s3.fr-par.scw.cloud`
9. `dvc push`
10. `aws s3api list-objects-v2 --bucket my-bucket` tu ensure object was pushed to bucket
