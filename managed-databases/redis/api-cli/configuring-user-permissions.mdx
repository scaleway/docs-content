When you create a Redis™ Database Instance, a single default database user is a created with [set permissions](/managed-databases/redis/reference-content/default-user-permissions).

You can specify a name for this user when you create a Database Instance via the Scaleway API or the CLI.

## Specifying a name for a user

Follow this procedure to [create a Database Instance](https://www.scaleway.com/en/developers/api/managed-database-redis/#quickstart).

A username and password for the default user are required to be included in your post request payload.

For a username to be valid, it must:
  - contain only letters, numbers, hyphens, dollars and underscores
  - be between 1 and 63 characters long
  - not start with `_rkv`
  - start with a letter
  - not be `scalewayadmin` or `metrics`

## Retrieving the default username

You can retrieve the default username via the API or the CLI.

<Tabs id="get database instance">
  <TabsTab label="API">
    Run the following curl command in a terminal to `GET` your Redis™ Database Instance of choice and retrieve the `default` username via the API. Replace `zone` and `cluster_id` in the path with the zone and cluster ID of your Database Instance of choice.

    ```bash
    curl -X GET \
    -H "X-Auth-Token: $SCW_SECRET_KEY" \
    "https://api.scaleway.com/redis/v1/zones/{zone}/clusters/{cluster_id}"
    ```

  </TabsTab>

  <TabsTab label="CLI">

    Run the following command in a terminal to `GET` information about your Redis™ Database Instance. Replace `<cluster_id>` with the ID of your Database Instance of choice.

    ```bash
    scw redis cluster get <cluster-id...>
    ```

  </TabsTab>
</Tabs>

In both cases, a response like the following is returned:

```json
{
"id": "string",
"name": "string",
"project_id": "string",
"status": "string",
"version": "string",
"endpoints": [],
"tags": [],

"updated_at": "string",
"tls_enabled": "boolean",
"cluster_settings": [],
"acl_rules": [],
"cluster_size": "integer",
"zone": "string",
"user_name": "string",
"upgradable_versions": []
}
```

  The example above is a simplified version of the response. `strings` and `boolean` are replaced with actual values in a real response.
</Message>

## Changing the default username

You can change the default user's name and password anytime via the API and the CLI.

<Tabs id="get database instance">
  <TabsTab label="API">
    Run the following curl command in a terminal to update your Redis™ Database Instance.

    ```bash
    curl -X PATCH \
    -H "X-Auth-Token: $SCW_SECRET_KEY" \
    -H "Content-Type: application/json" \
    -d '{}' \
    "https://api.scaleway.com/redis/v1/zones/{zone}/clusters/{cluster_id}"
    ```

  </TabsTab>

  <TabsTab label="CLI">

    Run the following command in a terminal to `GET` information about your Redis™ Database Instance. Replace `<cluster_id>` with the ID of your Database Instance of choice, or the ID of the main node for Redis™ Database Instances in cluster mode.

    ```bash
    scw redis cluster get <cluster-id...>
    ```

  </TabsTab>
</Tabs>
