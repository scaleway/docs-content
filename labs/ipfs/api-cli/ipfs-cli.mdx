You can leverage our IPFS pinning service to store permanently your data on the public IPFS network. You can do it via a CID (Content-ID) or an URL. You can [create](/labs/ipfs/api-cli/volumes-operations) and manage your volume via the [Kubo client](https://docs.ipfs.tech/install/command-line/) and the [IPFS Desktop](https://docs.ipfs.tech/install/ipfs-desktop/).

Kubo is the official implementation of IPFS, supported by [Protocol Labs](https://protocol.ai/), available on Windows, Linux and Mac.

## Requirements to use our service throught Kubo client

<Macro id="iam-requirements" />

<Message type="requirement">
    - You have an account and are logged into the [Scaleway Console](https://console.scaleway.com/)
    - You have generated your [API key](/identity-and-access-management/iam/how-to/create-api-keys/)
    - You have downloaded and installed the [IPFS Desktop](https://docs.ipfs.tech/install/ipfs-desktop/) or the [Kubo client](https://docs.ipfs.tech/install/command-line/)
    - You have create a [Volume](/labs/ipfs/api-cli/volumes-operations/)
</Message>

## Init and start your kubo client

Open a terminal and enter:

**CLI command**
```
ipfs init
```

then do:

**CLI command**
```
ipfs daemon
```

## Add Scaleway as a remote pinning service

You have to add Scaleway as a remote service in your local Kubo via its CLI. On another terminal enter:

**CLI command**
```
ipfs pin remote service add --service=scaleway https://fr-par.ipfs.labs.scw.cloud/<your-volume-id>/ <SECRET_KEY>
```

<Message type="note">
    As our service is available in several regions, you can use its different endpoints:
    - `https://pl-waw.ipfs.labs.scw.cloud`
    - `https://nl-ams.ipfs.labs.scw.cloud`
    - `https://fr-par.ipfs.labs.scw.cloud`
</Message>


## Pin a content by its CID

**CLI command**
```
ipfs pin remote add --service=scaleway QmaGNR5PedL8kiTUMtsPeFFodL9HnYBMuUqAEQrv6CQH4F
```

**Response**
```
CID:    QmaGNR5PedL8kiTUMtsPeFFodL9HnYBMuUqAEQrv6CQH4F
Name:
Status: pinned
```

<Message type="note">
    You can have four status: 'queued', 'pinning', 'pinned' or 'failed'.
</Message>

## Remove CID in Remote Service

**CLI command**
```
ipfs pin remote rm --service=scaleway --cid=QmaGNR5PedL8kiTUMtsPeFFodL9HnYBMuUqAEQrv6CQH4F
```
