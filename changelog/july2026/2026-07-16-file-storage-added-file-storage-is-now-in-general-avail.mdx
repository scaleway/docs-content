[File Storage](https://www.scaleway.com/fr/file-storage) is now in General Availability in the PAR region.

- For Kubernetes users, dynamically provision Persistent Volumes with ReadWriteMany access by referring to [this documentation page](/kubernetes/how-to/use-sfs-with-kubernetes/)

- For non Kubernetes users, attach multiple CPU or GPU instances to a shared and fully managed file system via the [Console, CLI](/file-storage/quickstart/) or [Terraform](https://registry.terraform.io/providers/scaleway/scaleway/latest/docs/resources/file_filesystem).

To reach the ideal level of performance for your file system use cases, also refer to [performance scaling](/file-storage/reference-content/performance-scaling/) and [compatible instance type selection](/file-storage/reference-content/file-system-instance-selection/) recommendations.
