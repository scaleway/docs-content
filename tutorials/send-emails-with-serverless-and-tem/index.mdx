In this tutorial, you will learn how to send emails with [Transactional Email](/managed-services/transactional-email/quickstart/) and [Serverless Functions](/serverless/functions/quickstart/) using a Terraform configuration file that builds your entire infrastructure.

<Macro id="iam-requirements" />
 
<Message type="requirement">
  - You have an account and are logged into the [Scaleway console.](https://console.scaleway.com)
  - You have a domain name
  - You have [downloaded and installed Terraform](https://developer.hashicorp.com/terraform/downloads)
</Message>

## Clone the GitHub repository

We have created a [GitHub repository](https://github.com/quantumsheep/scw-demo-tem) that contains all the code you need to deploy your infrastructure. Open a terminal and clone the repository to your local machine using the following command:

```shell
git clone https://github.com/quantumsheep/scw-demo-tem
```

Run the following command to access the `scw-tem-demo` directory:

```shell
cd scw-tem-demo
```

### Files and directories

In the `scw-tem-demo` directory, you will find three folders: `infra`, `signup-handler` and `super-product`.

- The `infra` folder holds `main.tf`, the Terraform configuration file used to deploy your infrastructure.
- The `signup-handler` folder contains the files that will configure your function. The `main.go` file is the entry point for your function.
- The `super-product` folder holds an `index.html.tftpl` file that contains HTML. This file is a template for the registration page. On a créé un bucket website sur lequel on poste le site. Ça crée la fonction et génère le template avant de le mettre dans le bucket, ça crée les records qu’il faut (mx, DKIM et spf) C’est un tftpl (terraform template) parce dedans il y a une valeur qu’on va remplir avec l’endpoint de la fonction qu’on va créer

Once the clone has completed,open a text editor of your choice and open the `Archive` folder.
