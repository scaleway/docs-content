## Overview

### What is Scaleway File Storage?

Scaleway File Storage allows you to keep data in an organized hierarchy of files and folders. This hierarchy can be shared across a network, so multiple compute instances can work on the same collection of files.

Refer to the [Quickstart documentation](/file-storage/quickstart/) to get started with Scaleway File Storage.

### What Scaleway products are compatible with File Storage?

Scaleway File Storage is designed to be used in conjunction with Scaleway Instances and Kubernetes Kapsule.

Refer to the dedicated documentation pages for comprehensive information on [how to add a file system to an Instance](/file-storage/quickstart/), and on [how to use File Storage with Kubernetes Kapsule](/kubernetes/how-to/use-sfs-with-kubernetes/).

## Offering and availability

### In which region can I provision file systems?

Scaleway File Storage is currently available in the PAR region. The AMS region will be available soon.

### How many file systems can I create?

During the **free** Public Beta (until May 31, 2026), the following quotas apply per Organization:

- Maximum number of file systems: 2
- Total provisioned storage size across all file systems: 100 GB

### How many file systems can I attach to an Instance?

The number of file systems you can attach to an Instance depends on the Instance type. Refer to the [dedicated documentation](/file-storage/reference-content/file-system-instance-selection/) for more information.

## Specifications

### What are the technical specifications of Scaleway File Storage?

Scaleway File Storage allows you to create file systems from 25 GB to 50 TB. [Performance scales linearly](/file-storage/reference-content/performance-scaling/) with the provisioned capacity.

### Do I need to provision a VPC for my file system?

No. Scaleway File Storage provides a built-in private network between file systems and Instances by design.

### Is File Storage compatible with NFS?

Scaleway File Storage is a file-sharing solution optimized for seamless communication between Instances and their hosts. The remote connection to the storage backend is fully managed by Scaleway and is transparent to the user. You do not need to manually configure or connect via the NFS protocol, and you benefit from the full range of file system features and high-performance capabilities without the administrative overhead of traditional network protocols.

## Security and resilience

### How does Scaleway ensure the protection of my data in File Storage?

Scaleway automatically encrypts all data at rest on physical disks and routes all transfers over a dedicated, isolated network managed by Scaleway, inaccessible to other users and external interference.

### How does Scaleway ensure data resilience in File Storage?

Scaleway ensures high availability and data durability by distributing your data across multiple independent storage nodes and disks within one Availability Zone (triple replication). This architecture prevents data loss from individual hardware failures and ensures that your file system remains accessible during routine maintenance or localized equipment issues.
