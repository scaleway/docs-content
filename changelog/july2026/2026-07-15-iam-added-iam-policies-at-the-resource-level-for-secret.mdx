title: IAM policies at the resource level for Secret Manager - Key Manager - IAM
status: added
date: 2026-07-15
category: security-identity
product: iam
---

With new conditions in IAM rules using resource attributes, access can now be restricted on Secret Manager, Key Manager, and IAM itself. For example, using these conditions, you can now grant access to:
- A secret or a secret folder on Secret Manager
- A key on Key Manager
- A user, SSH key, API key, application, policy or group on IAM.

[For more information, read the dedicated documentation.](https://www.scaleway.com/en/docs/iam/reference-content/understanding-resource-level-conditions/)
